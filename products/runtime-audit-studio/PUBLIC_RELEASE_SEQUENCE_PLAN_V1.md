# Runtime Audit Studio Public Release Sequence Plan V1

## 1. Purpose

This document defines the recommended release order for the public `Runtime Audit Studio` asset stack.

It exists to answer:

- what should happen after the first public asset decision is made?

## 2. Current Candidate Set

Current ready-but-gated assets:

- `products/runtime-audit-studio/PUBLIC_SUMMARY_V1.md`
- `products/runtime-audit-studio/PROFILE_DRAFT_V1.md`
- `products/runtime-audit-studio/OUTREACH_COPY_V1.md`

## 3. Default Recommended Release Order

Default order:

1. summary
2. profile
3. outreach

## 4. Why This Order Exists

Recommended logic:

- the summary is the cleanest standalone first-touch asset
- the profile becomes more useful once the product already has a public explanation layer
- outreach works best after the product has a stable public destination to point at

## 5. Primary Path

If the sponsor chooses the summary as the first released asset:

### Step 1

Use:

- `products/runtime-audit-studio/PUBLIC_SUMMARY_APPROVAL_SHEET_V1.md`

to decide `publish now` or `keep gated`.

### Step 2

If the summary is published:

- move the profile into the active next-release slot

### Step 3

After the profile is either published or intentionally kept gated:

- keep outreach as the next reusable support asset

### Step 4

Use outreach last:

- as a signal asset
- as a pointer to the stronger public assets

## 6. Fallback Paths

If the summary stays gated:

- keep it as the default first-release candidate
- treat the profile as the best fallback release candidate

If the sponsor wants a marketplace-facing surface first:

- profile may go first
- summary remains the cleaner follow-on explanation layer

If the sponsor wants the fastest outward signal first:

- outreach may go first
- but the summary should be prepared as the immediate follow-on destination

## 7. Risk Notes By Order

Summary first:

- best clarity
- lowest claim-drift risk

Profile first:

- good for marketplace surfaces
- weaker as the main first public explanation

Outreach first:

- fastest external signal
- highest risk of underspecifying the offer

## 8. Recommended Sponsor Interpretation

Use this plan like this:

- choose the first released asset once
- keep the other assets ready but gated
- do not improvise the release order each time

## 9. Recommended Next Step

First choose the first released asset in:

- `products/runtime-audit-studio/FIRST_RELEASE_DECISION_SHEET_V1.md`

Then use the chosen asset's own approval path to decide:

- publish now
- or keep ready but gated
