## Upwork Variant

### Title
Runtime Audit for Local LLM and AI Automation Systems

### Short Subtitle
I inspect your AI runtime as it actually runs, then tell you what is stable, what is brittle, and what to fix first.

### Overview
Most AI workflow setups look fine in demos and become unclear in long-running use.

Northbridge Systems reviews local LLM and agentic runtime setups with an operations-first lens. I do not sell AI hype. I read the evidence your system already produces: supervisor state, heartbeat, queue behavior, local model health, runtime logs, and approval boundaries.

You get a bounded audit pack that explains:

- what is healthy right now
- what is brittle or misleading
- where autonomy is safe
- where approval gates should stay
- what to fix first

### Good Fit

- Ollama or other local-model workflows
- Codex / Claude-style coding agents
- background jobs, supervisors, or schedulers
- multi-worker AI setups
- memory, retry, or recovery logic

### Client Inputs

- a short description of the runtime
- the main concern or failure mode
- logs, state files, or exported runtime artifacts
- clear scope boundaries

### CTA
Start with the Snapshot Runtime Audit if you want one bounded readout before expanding into larger work.
