# Runtime Audit Studio Marketplace Listing Approval Sheet V1

## Purpose

This sheet is for approving the marketplace-facing listing text for `Runtime Audit Studio`.

Target asset:

- `products/runtime-audit-studio/MARKETPLACE_LISTING_DRAFT_V1.md`

This is not the same as approving the service itself.  
It is the approval layer for how the service is presented on external platforms such as Upwork or Contra.

## Review Scope

Review these five things before publication:

1. message accuracy
2. boundary strength
3. marketplace fit
4. pricing/scope coherence
5. immediate publication decision

## Decision 1: Message Accuracy

Question:
Does the listing describe what Northbridge can actually do today, based on current runtime and product evidence?

Approve if:

- the listing matches the current `Runtime Audit Studio` offer
- it does not imply hidden implementation work
- it does not imply production certification
- it does not imply capabilities not yet verified

Do not approve if:

- it reads like a generic "AI automation expert" profile
- it promises more than the audit pack actually delivers
- it collapses review, implementation, and production support into one offer

Decision:

- `approve`
- `revise`
- `reject`

## Decision 2: Boundary Strength

Question:
Does the listing clearly protect the approval boundary and bounded-scope nature of the offer?

Approve if:

- the text says what the audit checks
- the text says what it does not do
- the text does not imply sponsor-free autonomy expansion
- the text does not blur audit and implementation

Do not approve if:

- it sounds like Northbridge will just "make the system autonomous"
- it weakens approval gates to sound more attractive
- it turns observation into guarantee

Decision:

- `approve`
- `revise`
- `reject`

## Decision 3: Marketplace Fit

Question:
Is the listing direct enough for a marketplace reader who will scan quickly?

Approve if:

- the headline is specific
- the subtitle is readable in one pass
- the good-fit section is concrete
- the CTA tells the client what to send first

Do not approve if:

- the listing sounds like internal documentation
- the strongest point is buried too low
- the platform variant text is too abstract to post

Decision:

- `approve`
- `revise`
- `reject`

## Decision 4: Pricing and Scope Coherence

Question:
Does the listing align with the current package framing and quote sheet?

Approve if:

- `Snapshot Runtime Audit` still reads as the default first engagement
- the listing does not imply larger included work than the quote sheet
- the language is consistent with the current package boundaries

Reference:

- `products/runtime-audit-studio/QUOTE_SHEET_V1.md`
- `products/runtime-audit-studio/SAMPLE_QUOTE_V1.md`

Decision:

- `approve`
- `revise`
- `reject`

## Decision 5: Immediate Publication

Question:
Should this exact listing be published now if the marketplace slot is available?

Options:

- `publish now`
- `publish after light edits`
- `keep gated`

Use `keep gated` if:

- the service is ready but the listing still sounds too broad
- sponsor wants a different first public asset first
- platform-specific trimming still needs to be done

## Recommended Current Position

Current recommendation:

- `message accuracy`: approve
- `boundary strength`: approve
- `marketplace fit`: revise
- `pricing and scope coherence`: approve
- `immediate publication`: publish after light edits

Reason:
The offer is already bounded and honest, but the listing still benefits from one more pass toward stronger platform-specific brevity.
