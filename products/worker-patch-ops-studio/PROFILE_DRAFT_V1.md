## 2. Headline Option

Evidence-first worker patch reviews for local AI teams that need controlled correction, not self-improving-agent hype

## 3. Short Profile Version

I review worker-agent failures, identify the repeated pattern, and turn that into one bounded patch proposal someone can actually approve.

The work is direct: read the evidence, name the failure, prepare a reversible patch, and keep live application gated instead of pretending the agents can safely rewrite themselves.

## 4. Medium Profile Version

`Worker Patch Ops Studio` is for teams that already run worker agents and need a clearer answer than "the prompt probably needs work."

This is a good fit when the real question is operational:

- which worker is actually failing
- what failure pattern keeps repeating
- what prompt-level change is worth testing
- how should that patch be reviewed before any live apply happens

My style is evidence-first and bounded. I do not sell autonomous self-improvement mythology as if it were disciplined engineering, and I do not hide live prompt mutation inside a vague improvement package.

The goal is a reviewable correction path, not hype and not a pile of raw eval artifacts with no decision structure.
