# Worker Education Program

## 1. Purpose

This document defines how `Northbridge Systems` actively educates workers instead of only generating and storing them.

The immediate aim is to improve worker prompts through structured training artifacts before editing live prompts.

## 2. Education Principle

Worker education should happen in five layers:

1. baseline prompt
2. lab iteration plan
3. training brief
4. prompt revision candidate
5. prompt patch proposal

This keeps education reversible and auditable.

## 3. Training Brief

A training brief should answer:

- what behavior is being improved
- why this improvement matters
- what good looks like
- what failure looks like
- what to inspect in the next evaluation pass

## 4. Prompt Revision Candidate

A prompt revision candidate should:

- preserve the worker's authority boundary
- preserve role identity
- add only the current training focus
- stay shorter than a full rewrite when possible

Live prompts should not be overwritten blindly.

## 5. Rotation Rule

Education should rotate across the roster.

Each unattended cycle should be able to create:

- one new worker-lab plan
- one training brief
- one prompt revision candidate
- one prompt patch proposal
- one president-facing summary

## 6. Promotion Rule

Training artifacts do not mean a worker is promoted.

Promotion still requires:

- evaluation bundle review
- promotion review decision
- president acceptance of a revised prompt candidate

## 7. Current Intent

The current education phase is:

- prompt-first
- document-heavy
- reversible
- comparison-friendly

The system should learn to teach workers before it learns to edit them aggressively.
