# Technical Brief Service Line V1

## 1. Purpose

This document defines the first concrete revenue offer for the current organization.

The first service line is:

- technical research briefs
- technical decision memos
- API and software documentation support

This line exists because it fits the current worker roster, stays reversible, and does not require immediate access to client production systems.

## 2. Service Owner

- execution owner: `Northbridge Systems`
- review and synthesis support: `Southgate Research`
- sponsor approval required for: external publication, pricing commitments, contracts, and client-specific sensitive data handling

## 3. What We Are Selling

We are not selling "autonomous AI magic."

We are selling bounded technical thinking and documentation in a form that is easy for a client to review and easy for the sponsor to supervise.

Core deliverables:

- technical research brief
- technical decision memo
- API documentation cleanup or first-pass draft
- software documentation structure and rewrite support

## 4. Package Shapes

### Package A: Fast Technical Brief

Best for:

- a focused technical question
- a quick compare-and-recommend task
- a small API, library, or tooling choice

Deliverable shape:

- 2 to 4 pages
- problem framing
- option comparison
- recommendation
- unresolved risks

Internal draft pricing band:

- `8,000 JPY` to `20,000 JPY`

### Package B: Decision Memo

Best for:

- architecture or tooling decisions
- migration choices
- vendor or framework comparison
- implementation planning before build work

Deliverable shape:

- 4 to 8 pages
- context and goals
- evaluated options
- recommendation with rationale
- implementation notes
- risks and follow-up actions

Internal draft pricing band:

- `20,000 JPY` to `50,000 JPY`

### Package C: Documentation Support

Best for:

- API docs cleanup
- onboarding docs cleanup
- internal developer guide rewrites
- feature or workflow documentation

Deliverable shape:

- revised documentation set or structured draft
- change summary
- known gaps and follow-up list

Internal draft pricing band:

- `15,000 JPY` to `60,000 JPY`

## 5. What This Service Line Explicitly Does Not Include

- unsupervised production access
- client account creation
- direct deployment work by default
- financial operations
- security-sensitive actions without separate approval
- guaranteed legal, tax, or compliance advice

If the client request crosses these boundaries, the work must be reframed, split, or rejected.

## 6. Internal Worker Mapping

- `Compass`: source comparison, option mapping, uncertainty notes
- `Quill`: compression, structure, final readable draft shape
- `Ledger`: contradiction check, consistency check, unsupported-claim detection
- `Forge`: code examples, repo-backed snippets, implementation notes
- `Lantern`: continuity checks, stale-state detection, delivery checklist support

## 7. Standard Delivery Promise

Each paid brief should contain:

- the client question in plain terms
- the evaluated context
- the strongest recommendation
- the most important tradeoffs
- a clear list of unresolved risks
- a next-action section

The client should not need a second meeting just to understand the output.

## 8. Approval Boundary

This service line can be prepared internally without sponsor interruption.

The following still require sponsor approval:

- sending a quote externally
- accepting a client engagement
- using paid external APIs beyond current policy
- handling sensitive client material
- agreeing to delivery dates or refunds

The next operational step is to pair this offer with:

- a sponsor-facing intake form
- a reusable delivery template
- a sponsor-facing quote sheet
- a bounded internal delivery checklist
