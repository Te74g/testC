# Sponsor Approval Manual

## 1. Purpose

This document defines how `Northbridge Systems` and the `Claude-side company` request sponsor approval.

The sponsor is the final authority for financial, external, destructive, and sensitive actions.

## 2. Approval Categories

Sponsor approval is required for:

- subscriptions
- hardware purchases
- stock purchases
- paid API expansion
- new online services
- external publication
- destructive file actions with business impact
- production deployment
- use of sensitive data

## 3. Approval Request Standard

Every approval request must contain:

- request title
- requesting company
- objective
- exact action requested
- expected benefit
- estimated cost
- risks
- alternatives considered
- rollback or exit plan
- deadline

## 4. Approval Levels

### Level 1

Low-risk, low-cost expansion.

Examples:

- small API budget increase
- temporary tool trial
- limited compute allocation

### Level 2

Moderate cost or moderate impact.

Examples:

- monthly plan upgrade
- dedicated hardware proposal
- longer-running worker pool

### Level 3

High-cost, high-impact, or irreversible.

Examples:

- stock purchase
- multi-month hardware financing
- external business-facing launch

## 5. Decision Output

The sponsor may respond with:

- approved
- approved with conditions
- denied
- request revision
- ask for more evidence

## 6. No-Implied Approval Rule

Silence is not approval.

If the sponsor has not explicitly approved the request, the companies must treat the request as pending.

## 7. Benefit Verification

After approval and execution, the requesting company must report:

- what was done
- actual cost
- actual benefit
- variance from plan
- whether the approval should be considered successful

This keeps future approvals grounded in evidence.
