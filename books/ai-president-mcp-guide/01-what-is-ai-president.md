# AI社長組織とは何か

## AIに「チーム」を持たせるという発想

Claude CodeやChatGPTを使ったことがあるなら、こんな不満を感じたことはないだろうか。

- 「1つのAIに全部やらせると、途中で方向を見失う」
- 「調査と実装を同時にやらせると、どっちも中途半端になる」
- 「AIの出力を別のAIに渡して処理させたいけど、コピペが面倒」

人間の組織では、これを「役割分担」で解決している。社長が方針を決め、リサーチャーが調べ、エディターがまとめる。AIでも同じことができたら？

本書では、**Claude Code を「社長」、ローカルLLM（Ollama + Qwen 2.5 Coder 14B）を「部下ワーカー」として、実際に動くAI組織**を作る。社長がメッセージを送ると、部下が自動的にタスクを拾い、ローカルGPUで推論し、結果を返してくる。

これは空想ではない。本書のコードは、筆者が実際に構築・稼働させたシステムそのものだ。

## 全体アーキテクチャ

```
┌─────────────────────────────────────────────┐
│  スポンサー（人間）                            │
│  └─ 指示・承認・最終レビュー                    │
└──────────────┬──────────────────────────────┘
               │
┌──────────────▼──────────────────────────────┐
│  AI社長 (Claude Code)                        │
│  ・戦略立案・タスク設計・品質レビュー             │
│  ・MCP経由でワーカーに指示                      │
└──────────────┬──────────────────────────────┘
               │ MCPメッセージバス
               │ (SQLite + JSON-RPC 2.0)
       ┌───────┴───────┐
       ▼               ▼
┌──────────────┐ ┌──────────────┐
│ W-03 Researcher │ │ W-04 Editor   │
│ (Compass)       │ │ (Quill)       │
│ Ollama          │ │ Ollama        │
│ Qwen 2.5 14B   │ │ Qwen 2.5 14B  │
│ ・調査・分析     │ │ ・要約・整形   │
└──────────────┘ └──────────────┘
     RTX 4070 Ti SUPER (16GB VRAM)
```

ポイントは3つ。

**1. MCPメッセージバス（本書で自作する）**

MCP (Model Context Protocol) は、AIツール統合のためのオープンプロトコルだ。JSON-RPC 2.0ベースで、stdioを通じてAIクライアントとツールサーバーが通信する。

本書では、このMCPを使って「エージェント間メッセージバス」を作る。社長が `send_message` でタスクを送り、ワーカーが `read_inbox` でタスクを受け取る。SlackやTeamsのような仕組みを、AI専用に、SQLiteで永続化して作る。

**2. ローカルLLMワーカー（GPUで推論、API代ゼロ）**

部下ワーカーはクラウドAPIではなく、手元のGPUで動く。RTX 4070 Ti SUPER（16GB VRAM）があれば、Qwen 2.5 Coder 14Bが快適に動作する。API代が毎月かかるのではなく、電気代だけで動く。

**3. プロンプトによる役割制御**

各ワーカーには専用のシステムプロンプトがある。Researcherは「3つの選択肢を提示し、推奨と不確実性を明記する」、Editorは「次のアクションと未解決リスクで締める」。これにより、同じLLMモデルでも役割に応じた異なる出力が得られる。

## 本書で作れるもの

本書を読み終えると、以下が手元で動く状態になる。

- MCPメッセージバスサーバー（TypeScript、8ツール、SQLite永続化）
- ワーカーランナー（Node.js、Ollamaポーリング、自動応答）
- ワーカープロンプト一式（Researcher、Editor）
- `.mcp.json` によるClaude Code統合

所要時間は、環境構築込みで **2〜3時間** を見てほしい。Node.js と GPU（VRAM 12GB以上推奨）が必要だ。

## 想定読者

- Claude Code / Cursor / Cline などのAIコーディングツールを使っている人
- ローカルLLMに興味があるが、何に使えるか分からない人
- MCPサーバーを自作してみたい人
- AIエージェントの「マルチエージェント構成」を手元で試したい人

前提知識として、TypeScriptの基本的な読み書きと、ターミナル操作ができれば十分だ。
