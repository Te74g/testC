# Sample Technical Brief Quote V1

## 1. Purpose

This document is a sponsor-facing example quote for the first documentation-based revenue line.

It is an internal sample.

It exists to show how a real quote should look before any external client-facing quote is approved and sent.

## 2. Example Client Request

Client asks:

- "We need a short recommendation on whether to run our Windows unattended AI workflow through Task Scheduler or a user-space launcher."

What they want:

- a short written recommendation
- clear tradeoffs
- a practical next-step section
- no implementation work yet

## 3. Recommended Package

- package: `Package A - Fast Technical Brief`
- reason: the question is narrow, decision-shaped, and bounded enough to finish without external credentials or live implementation

## 4. Example Quote

- quote title: Windows unattended runtime decision brief
- scope summary: compare two runtime ownership patterns for a Windows unattended AI workflow and recommend one approach for a small operator-run environment
- deliverable:
  - 1 technical brief
  - target length: `3 to 4 pages`
  - sections:
    - question framing
    - option comparison
    - recommendation
    - risks
    - next actions
- turnaround: `2 business days`
- price: `15,000 JPY`
- included revisions:
  - one bounded revision pass for wording, structure, and emphasis
- excluded items:
  - live implementation
  - scheduler setup
  - production deployment
  - external account work
  - open-ended consultation beyond the quoted question
- sponsor approval status:
  - internal sample only
  - not yet approved for external sending

## 5. Why This Price Fits

- the scope is narrow but still requires real comparison and synthesis
- the output has to be concise and decision-ready, not just a rough note
- the delivery does not include live environment changes, which keeps risk and cost lower than an implementation package

## 6. Example External Wording

Draft quote language:

> I can prepare a short technical decision brief comparing Task Scheduler and a user-space launcher for a Windows unattended AI workflow. The deliverable would be a 3 to 4 page brief with recommendation, tradeoffs, risks, and next steps. Turnaround would be 2 business days. Price: 15,000 JPY. One bounded revision pass for wording and structure is included. Live implementation, deployment, and open-ended consulting are not included in this quote.

## 7. Internal Delivery Mapping

- `Compass`: option comparison and uncertainty notes
- `Quill`: final brief structure and readable draft
- `Ledger`: contradiction and unsupported-claim check
- `Forge`: concrete implementation notes where useful
- `Lantern`: delivery checklist and continuity check
