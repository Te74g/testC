# First Presidents' Meeting

## 1. Purpose

This is the first formal meeting pack for:

- the president of `Northbridge Systems`
- the president of the `Claude-side company`

The purpose is to move from policy documents into the first operational setup.

## 2. Required Outcomes

This meeting should produce decisions on:

1. the Claude-side company name, if Claude Code is ready to choose it
2. the first worker roster
3. the first safe execution loop
4. the first candidate money-making task categories
5. the minimum sponsor approval thresholds for pilot execution

## 3. Pre-Read

Review before the meeting:

- `COMPANY_FOUNDATION.md`
- `INTERCOMPANY_PROTOCOL.md`
- `WORKER_CREATION_MANUAL.md`
- `PROCESS_ORDER.md`
- `memory/DURABLE_MEMORY.md`
- `memory/ACTIVE_CONTEXT.md`

## 4. Agenda

### Agenda Item 1: Company Identity

Questions:

- Has Claude Code chosen the permanent company name?
- If not, does the placeholder remain acceptable for now?

Desired output:

- company naming status

### Agenda Item 2: Operational Division

Questions:

- Which work should stay with `Northbridge Systems`?
- Which work should stay with the Claude-side company?
- Which work should be shared?

Desired output:

- first operating split

### Agenda Item 3: First Worker Roster

Questions:

- Which small worker set is enough to start?
- Which workers should not exist yet?

Desired output:

- initial roster approval

### Agenda Item 4: First Safe Execution Loop

Questions:

- What task can be run locally, safely, and repeatedly?
- How will quality be verified?
- What is the escalation rule?

Desired output:

- one sandboxed operating loop

### Agenda Item 5: First Monetization Hypotheses

Questions:

- Which task categories might create real value first?
- Which ones are too risky or premature?

Desired output:

- shortlist of pilot categories

## 5. Decision Record Template

- Decision:
- Owner:
- Why:
- Constraint:
- Sponsor approval needed: yes/no

## 6. Mandatory Non-Decisions

The meeting should explicitly avoid doing these prematurely:

- hardware purchase
- stock purchase
- open-ended worker creation
- unrestricted external API spend
- irreversible automation

## 7. Post-Meeting Deliverables

After this meeting, the expected artifacts are:

- approved worker roster
- first execution loop document
- monetization candidate list
- sponsor approval requests if needed
