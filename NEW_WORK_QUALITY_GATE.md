# New Work Quality Gate

## 1. Purpose

This document defines the absolute rule for starting net-new work.

The goal is to stop weak or under-researched ideas from quietly entering the active bot loop.

## 2. Net-New Work Definition

Treat the following as new work:

- a new task category
- a new money-making experiment
- a new automation type
- a new worker class
- a new external integration
- a new runtime capability
- a new approval-mode widening step

Routine continuation of already-defined work is not net-new work.

## 3. Mandatory Research Rule

Before starting net-new work, perform web-based multi-angle analysis.

The analysis should try to cover:

- technical feasibility
- safety and abuse risk
- operational cost
- reversibility
- alternatives

Do not start net-new work from intuition alone.

## 4. Self-Check Score

After the analysis, score the proposal from 0 to 100.

Use five axes:

- problem clarity
- evidence quality
- operational safety
- reversibility
- expected value

Each axis should be scored from 0 to 20.

Total score:

- `0 to 100`

## 5. Decision Bands

### `80 to 100`

Result:

- good enough to continue
- bot-assisted continuation is allowed
- move the task into the active work loop

### `61 to 79`

Result:

- continue through a changed approach
- do additional web research
- restructure the task
- rescore while staying in the bot-managed loop

This band is treated as a rebuild band, not a stop band.

### `40 to 60`

Result:

- perform broader multi-angle web research
- reconstruct the task from a cleaner frame
- continue through a materially changed approach
- do not use the old framing

This band is still bot-continuable, but only after reframing.

### `0 to 39`

Result:

- interrupt
- do not continue
- do not hand it to the bot

If the idea returns later, it must restart from research.

## 6. Infinite Work Rule

The system should not run out of work.

If routine backlog is low, generate candidate work.

But generated work must still pass the new-work quality gate before entering active execution.

Infinite work is allowed.

Infinite low-quality work is not allowed.

Work scoring `40+` should remain in managed circulation, even if the approach must change.

## 7. Bot Rule

Bots may continue:

- approved routine work
- work that already passed the quality gate
- work scoring `40+` that is being actively reframed under the quality-gate rules

Bots may not autonomously promote weak new ideas into active execution.

## 8. Sponsor Interaction Rule

When a genuinely new direction appears, the expected sequence is:

1. web-based multi-angle analysis
2. self-check score
3. band-based decision
4. only then bot continuation if the score is high enough

For `40+` scores, bot continuation remains allowed, but the bot must follow the required rebuild path for that score band.

## 9. Recording Rule

For net-new work, record:

- the proposal
- the score
- the decision band
- whether it was continued, rebuilt, or interrupted

Short records belong in memory.

Use this template when needed:

- `templates/new_work_scorecard.md`
