# Technical Brief Quote Sheet V1

## 1. Purpose

This document is the sponsor-facing pricing and quoting guide for the first documentation-based revenue line.

It is not an automatic public price list.

It is an internal quoting sheet used before any sponsor-approved external offer is sent.

## 2. Quote Rules

- all external quotes still require sponsor approval
- use the smallest package that honestly fits the request
- do not quote implementation work under a documentation package
- if the request needs credentials, production access, or high-risk external action, stop and escalate

## 3. Standard Packages

### Package A: Fast Technical Brief

- ideal scope: one narrow technical question
- output: 2 to 4 pages
- turnaround target: 1 to 3 days
- internal quote band: `8,000 JPY` to `20,000 JPY`

Use this when:

- the client needs a compare-and-recommend output
- the scope is narrow enough to finish without external integration
- the main value is clarity, not implementation

### Package B: Decision Memo

- ideal scope: one technical decision with multiple tradeoffs
- output: 4 to 8 pages
- turnaround target: 2 to 5 days
- internal quote band: `20,000 JPY` to `50,000 JPY`

Use this when:

- the client has a framework, architecture, or migration choice
- the output needs rationale, risks, and next actions
- implementation is still out of scope

### Package C: Documentation Support

- ideal scope: one bounded documentation cleanup or structured draft
- output: one revised doc set plus change summary
- turnaround target: 2 to 7 days
- internal quote band: `15,000 JPY` to `60,000 JPY`

Use this when:

- the client already has rough documentation
- the main need is structure, clarity, completeness, or rewrite quality
- no privileged system access is required

## 4. Quote Modifiers

### Increase price when:

- the source material is messy
- the domain is specialized
- the client wants English and Japanese versions
- the brief needs code examples or repo-backed examples
- the expected reader is executive and technical at the same time

### Keep price lower when:

- the question is very narrow
- source materials are already organized
- the expected output is short
- the deadline is flexible

## 5. Do Not Quote Under This Service Line If:

- the client really wants live implementation
- the request needs API keys, admin access, or production systems
- the problem is mostly legal or compliance advice
- the client is asking for open-ended consulting with no bounded question

In these cases:

- reframe the work
- split the work
- or decline the work

## 6. Sponsor Review Checklist Before Sending A Quote

- package fit is correct
- scope is bounded
- no hidden implementation commitment exists
- timeline is realistic
- sensitive-data handling is understood
- quote wording matches the actual deliverable
