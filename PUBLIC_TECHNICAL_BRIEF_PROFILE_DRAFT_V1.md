This document is a short external-facing profile draft for the technical brief service line.

It is still sponsor-review material, not auto-approved public copy.

## 2. Headline Option

Technical briefs, decision memos, and documentation cleanup for messy software questions

## 3. Short Profile Version

I help turn fuzzy technical questions into clear written outputs.

That includes technical briefs, decision memos, API documentation cleanup, and developer documentation rewrites. My work is bounded and reviewable: clear framing, explicit tradeoffs, practical next steps, and no pretending documentation is implementation.

## 4. Medium Profile Version

I work on technical briefs, decision memos, and documentation support for software and developer-facing work.

This is a good fit when the real need is clarity before implementation:

- compare tools, frameworks, or APIs
- frame a migration or architecture choice
- clean up rough API docs
- rewrite internal developer documentation

My style is direct and low-hype. I try to make the recommendation clear, the tradeoffs explicit, and the next step easy to understand.

If the work expands into production implementation, deployment, or open-ended consulting, that should be scoped separately rather than hidden inside a documentation quote.

## 5. Best-Fit Keywords

- technical brief
- decision memo
- API documentation
- software documentation
- developer documentation
- architecture comparison
- tooling evaluation
- migration planning

## 6. Best-Fit Client Situations

- "We need a short recommendation before we implement."
- "We have rough docs, but they are too messy to use."
- "We need the real tradeoffs written down clearly."
- "We need a bounded first deliverable, not an open-ended consulting arrangement."

## 7. Soft Boundary Version

I can help frame implementation decisions and prepare clear technical documentation, but I do not hide live implementation work inside a documentation quote.

## 8. Start-Here Version

If useful, start with one small scoped brief first.

That usually creates:

- a concrete output
- a bounded price
- a clear decision artifact

before anything larger is discussed.
